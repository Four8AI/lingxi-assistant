# Web服务404错误说明

## 常见404错误

### 1. `/%40vite/client` 404错误

**错误信息**:
```
INFO:     ::1:62616 - "GET /%40vite/client HTTP/1.1" 404 Not Found
```

**原因**:
这个请求通常由浏览器扩展或开发工具自动发出，例如：
- React DevTools
- Vue.js DevTools
- 其他前端开发工具

这些工具会尝试检测页面是否使用Vite等构建工具，并自动发送探测请求。

**解决方案**:
- **无需处理** - 这不是我们应用的问题
- 可以在浏览器中禁用相关扩展
- 或者忽略这些404日志

### 2. `/favicon.ico` 404错误

**错误信息**:
```
INFO:     ::1:63955 - "GET /favicon.ico HTTP/1.1" 404 Not Found
```

**原因**:
浏览器自动请求网站图标（favicon）。

**解决方案**:
已添加favicon文件：
- 文件路径: `web/static/favicon.svg`
- HTML中已添加: `<link rel="icon" type="image/svg+xml" href="/static/favicon.svg">`

## 如何减少404日志

### 方法1: 配置日志过滤器

在`config.yaml`中添加日志过滤器：

```yaml
logging:
  level: "INFO"
  file_path: "logs/assistant.log"
  max_file_size_mb: 10
  backup_count: 5
  format: "%(asctime)s - %(levelname)s - %(message)s"
  
  # 忽略特定404错误
  ignore_patterns:
    - "/%40vite/client"
    - "/favicon.ico"
```

### 方法2: 添加自定义路由

在`web/fastapi_server.py`中添加：

```python
from fastapi.responses import JSONResponse

@app.get("/favicon.ico")
async def favicon():
    """返回空响应，避免404错误"""
    return JSONResponse({}, status_code=204)

@app.get("/{path:path}")
async def catch_all(path: str):
    """捕获所有未匹配的路由"""
    if path.startswith("%40"):
        return JSONResponse({}, status_code=204)
    raise HTTPException(status_code=404, detail="Not Found")
```

### 方法3: 使用日志过滤器（推荐）

修改`lingxi/utils/log_filters.py`：

```python
class Ignore404Filter(logging.Filter):
    """忽略特定404请求的日志"""

    def filter(self, record):
        message = record.getMessage()
        
        # 忽略浏览器扩展的探测请求
        if "404 Not Found" in message:
            if "/%40vite/client" in message or "/favicon.ico" in message:
                return False
        
        return True
```

然后在`lingxi/utils/logging.py`中应用：

```python
from lingxi.utils.log_filters import Ignore404Filter

ignore_404_filter = Ignore404Filter()
console_handler.addFilter(ignore_404_filter)
file_handler.addFilter(ignore_404_filter)
```
