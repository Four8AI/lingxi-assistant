# PySide6 Agent 界面程序系统详细设计文档

## 一、项目概述
### 1.1 项目背景
本项目旨在开发一个基于 PySide6 的 Agent 服务管理界面，用于管理 AI Agent 的任务执行、日志查看、依赖管理和报告预览等核心功能，提升 Agent 服务的可观测性和可操作性。

### 1.2 核心目标
- 提供清晰的任务管理面板，支持任务创建、执行和状态监控
- 实现实时日志展示和依赖版本管理功能
- 支持 Markdown 格式的报告预览和编辑
- 提供多窗口/多面板的布局管理，提升操作效率

### 1.3 技术选型
- **GUI 框架**：PySide6（LGPL 授权，跨平台支持）
- **布局管理**：QSplitter + QVBoxLayout/QHBoxLayout
- **日志展示**：QTextEdit + QThread 异步读取
- **报告预览**：QTextEdit + Markdown 渲染（或 QWebEngineView）
- **依赖管理**：subprocess + QTableWidget

## 二、系统架构设计
### 2.1 整体架构
采用 **MVC（Model-View-Controller）** 架构，将界面、业务逻辑和数据分离，提升可维护性：
- **Model**：负责管理 Agent 任务、依赖版本、日志数据等核心数据
- **View**：基于 PySide6 实现的界面组件，包括任务列表、日志面板、报告预览等
- **Controller**：处理用户交互，协调 Model 和 View 之间的数据流转

### 2.2 模块划分
| 模块名称 | 核心职责 | 关键组件 |
|----------|----------|----------|
| 主窗口模块 | 管理整体布局和窗口生命周期 | QMainWindow, QSplitter |
| 任务管理模块 | 展示任务列表、任务状态和操作按钮 | QListWidget, QPushButton |
| 日志与终端模块 | 实时展示日志、执行终端命令 | QTextEdit, QLineEdit, QThread |
| 依赖管理模块 | 展示依赖版本、支持版本修复 | QTableWidget, QMessageBox |
| 报告预览模块 | 预览和编辑 Markdown 格式报告 | QTextEdit, QWebEngineView |
| 通信模块 | 与 Agent 服务进程交互 | subprocess, QProcess |

## 三、核心模块详细设计

### 3.1 主窗口模块
#### 3.1.1 布局设计
采用三栏式布局，通过 QSplitter 实现可拖拽调整：
- **左侧栏**：任务列表 + 任务操作按钮
- **中间栏**：日志/终端面板 + 依赖管理面板
- **右侧栏**：报告预览/编辑器

#### 3.1.2 关键组件
- `AgentMainWindow(QMainWindow)`：主窗口类，管理所有子面板
- `QSplitter`：实现三栏布局，支持宽度调整
- `QMenuBar`：提供文件、编辑、视图等菜单操作
- `QStatusBar`：显示当前状态信息（如任务执行中、依赖状态）

### 3.2 任务管理模块
#### 3.2.1 功能设计
- 展示任务列表，包含任务名称、状态、完成时间
- 支持任务创建、启动、停止、删除操作
- 任务状态实时更新（运行中、已完成、失败）

#### 3.2.2 关键组件
- `TaskListWidget(QListWidget)`：任务列表组件，自定义任务项样式
- `TaskItemWidget(QWidget)`：单个任务项，包含任务名称、状态图标、操作按钮
- `TaskManager(QObject)`：任务管理逻辑，维护任务状态和生命周期

### 3.3 日志与终端模块
#### 3.3.1 功能设计
- 实时展示 Agent 服务的日志输出
- 支持终端命令执行（如 `pip show`、`pip install`）
- 日志自动滚动，支持清空、保存到文件

#### 3.3.2 关键组件
- `LogTextEdit(QTextEdit)`：只读日志展示框，支持自动滚动
- `TerminalInput(QLineEdit)`：终端命令输入框
- `LogThread(QThread)`：异步读取子进程输出，避免界面卡顿
- `CommandExecutor(QObject)`：执行终端命令，处理输出和错误

### 3.4 依赖管理模块
#### 3.4.1 功能设计
- 展示当前 Python 环境的依赖版本
- 标记不兼容的依赖版本，提供修复方案
- 支持一键执行版本修复命令

#### 3.4.2 关键组件
- `DependencyTable(QTableWidget)`：依赖版本表格，包含包名、版本、状态
- `DependencyChecker(QObject)`：检查依赖兼容性，生成修复方案
- `FixButton(QPushButton)`：触发依赖修复操作

### 3.5 报告预览模块
#### 3.5.1 功能设计
- 预览 Markdown 格式的 Agent 输出报告
- 支持报告编辑和保存
- 提供报告导出功能（如 PDF、HTML）

#### 3.5.2 关键组件
- `ReportPreview(QTextEdit)`：Markdown 预览/编辑框
- `MarkdownRenderer(QObject)`：渲染 Markdown 内容为富文本
- `ExportButton(QPushButton)`：触发报告导出操作

## 四、核心流程设计

### 4.1 任务执行流程
1. 用户点击「新任务」按钮，创建任务
2. 任务管理器启动 Agent 子进程
3. 日志线程实时读取子进程输出，更新日志面板
4. 任务完成后，更新任务状态，生成报告
5. 报告自动加载到预览面板

### 4.2 依赖修复流程
1. 依赖检查器扫描当前环境依赖版本
2. 标记不兼容的依赖，生成修复方案
3. 用户点击修复按钮，执行 `pip install` 命令
4. 命令执行完成后，重新检查依赖兼容性
5. 显示修复结果，更新依赖表格

### 4.3 报告预览流程
1. Agent 生成 Markdown 格式报告
2. 报告加载到预览面板，自动渲染为富文本
3. 用户可编辑报告内容，保存或导出报告

## 五、界面设计规范

### 5.1 颜色与主题
- **主色调**：深灰色 (#2E2E2E) 作为背景，突出专业感
- **强调色**：绿色 (#4CAF50) 表示成功，红色 (#F44336) 表示错误
- **文本色**：白色 (#FFFFFF) 用于主文本，浅灰色 (#BDBDBD) 用于次要文本

### 5.2 字体与排版
- **主字体**：Microsoft YaHei / Segoe UI，12pt
- **标题字体**：14pt，加粗
- **代码字体**：Consolas / Courier New，10pt

### 5.3 交互设计
- 所有操作按钮提供悬停效果和点击反馈
- 任务状态变化时，通过图标和颜色变化提示用户
- 日志面板支持右键菜单（复制、清空、保存）
- 报告预览支持快捷键（Ctrl+S 保存，Ctrl+E 导出）

## 六、关键技术实现

### 6.1 异步日志读取
```python
class LogThread(QThread):
    log_signal = Signal(str)
    
    def __init__(self, process):
        super().__init__()
        self.process = process
        self.running = True
    
    def run(self):
        while self.running and self.process.poll() is None:
            line = self.process.stdout.readline()
            if line:
                self.log_signal.emit(line.decode('utf-8', errors='ignore'))
    
    def stop(self):
        self.running = False
```

### 6.2 依赖版本检查
```python
class DependencyChecker(QObject):
    def check_compatibility(self):
        import pkg_resources
        dependencies = {
            "requests": ">=2.25.0",
            "chardet": ">=3.0.2, <6.0.0",
            "urllib3": ">=1.21.1"
        }
        incompatible = []
        for pkg, req in dependencies.items():
            try:
                pkg_resources.require(req)
            except pkg_resources.VersionConflict as e:
                incompatible.append({
                    "package": pkg,
                    "current": e.dist.version,
                    "required": req,
                    "fix": f"pip install {pkg}{req.replace('>=', '>=').replace('<', '<')}"
                })
        return incompatible
```

### 6.3 Markdown 报告渲染
```python
class MarkdownRenderer(QObject):
    def render(self, markdown_text):
        import markdown
        html = markdown.markdown(markdown_text, extensions=['tables', 'fenced_code'])
        return f"""
        <html>
        <head>
            <style>
                body {{ font-family: Microsoft YaHei; background-color: #2E2E2E; color: #FFFFFF; }}
                table {{ border-collapse: collapse; width: 100%; }}
                th, td {{ border: 1px solid #BDBDBD; padding: 8px; text-align: left; }}
                th {{ background-color: #424242; }}
            </style>
        </head>
        <body>{html}</body>
        </html>
        """
```

## 八、风险与应对
| 风险点 | 影响 | 应对措施 |
|--------|------|----------|
| 日志数据量大导致界面卡顿 | 界面无响应 | 使用 QThread 异步读取，限制日志缓存大小 |
| 依赖版本冲突 | Agent 服务无法启动 | 启动前自动检查依赖兼容性，提供一键修复 |
| 跨平台兼容性问题 | 界面显示异常 | 使用 Qt 原生组件，避免平台特定代码 |
