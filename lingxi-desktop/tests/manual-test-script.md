# 手动测试脚本

## 测试环境准备

### 前置条件
- Node.js v18+ 已安装
- 项目依赖已安装 (`npm install`)
- 后端服务已启动（如适用）

### 测试环境
- **Electron 模式**: `npm run electron:dev`
- **Web 模式**: `npm run dev` (浏览器访问 http://localhost:5173)

---

## 1. 聊天功能测试

### 1.1 发送文本消息
**步骤**:
1. 打开应用
2. 在聊天输入框中输入测试文本："你好，这是一个测试消息"
3. 点击发送按钮或按 Enter 键

**预期结果**:
- 消息立即显示在聊天窗口中
- 消息气泡显示在右侧（用户消息）
- 输入框清空

### 1.2 接收 AI 回复
**步骤**:
1. 发送一条消息后等待
2. 观察 AI 回复

**预期结果**:
- AI 回复在合理时间内出现（<3 秒）
- 回复内容相关且合理
- 消息气泡显示在左侧（AI 消息）

### 1.3 流式输出显示
**步骤**:
1. 发送一个需要较长回复的问题（如"请解释量子力学"）
2. 观察回复显示过程

**预期结果**:
- 回复以流式方式逐字显示
- 显示流畅，无明显卡顿
- 完整回复最终显示

### 1.4 停止生成
**步骤**:
1. 发送一个长问题触发长回复
2. 在回复生成过程中点击"停止"按钮

**预期结果**:
- 生成立即停止
- 已生成的内容保留
- 按钮状态恢复为发送

### 1.5 清除对话历史
**步骤**:
1. 进行多轮对话
2. 点击"清除对话"按钮
3. 确认清除操作

**预期结果**:
- 所有对话消息被清除
- 聊天窗口变为空白
- 会话状态重置

### 1.6 代码块渲染
**步骤**:
1. 发送请求："请写一个 Python 的 Hello World 程序"
2. 观察回复中的代码块

**预期结果**:
- 代码以代码块形式显示
- 语法高亮正确
- 代码格式保持完整
- 有复制按钮（如有）

### 1.7 Markdown 渲染
**步骤**:
1. 发送请求："请用 Markdown 格式列出 3 个要点"
2. 观察回复格式

**预期结果**:
- 标题正确渲染
- 列表正确显示
- 粗体/斜体正确显示
- 链接可点击

---

## 2. 会话管理测试

### 2.1 创建新会话
**步骤**:
1. 点击"新建会话"按钮
2. 输入会话名称（可选）
3. 确认创建

**预期结果**:
- 新会话创建成功
- 聊天窗口清空
- 会话列表中出现新会话

### 2.2 切换会话
**步骤**:
1. 创建多个会话
2. 在会话列表中点击不同会话

**预期结果**:
- 聊天内容正确切换
- 当前会话高亮显示
- 历史消息正确加载

### 2.3 重命名会话
**步骤**:
1. 右键点击会话或点击编辑按钮
2. 选择"重命名"
3. 输入新名称并确认

**预期结果**:
- 会话名称更新
- 更改持久化保存

### 2.4 删除会话
**步骤**:
1. 右键点击会话或点击删除按钮
2. 确认删除操作

**预期结果**:
- 会话从列表中移除
- 相关数据被清除
- 自动切换到其他会话

### 2.5 导出会话
**步骤**:
1. 选择要导出的会话
2. 点击"导出"功能
3. 选择导出格式（JSON/Markdown）
4. 选择保存位置

**预期结果**:
- 文件成功导出
- 内容格式正确
- 包含完整的对话历史

**步骤**:
1. 发送消息并计时
2. 记录到收到回复的时间

**预期结果**:
- 首字显示 < 1 秒

### 6.3 内存占用
**步骤**:
1. 打开任务管理器/活动监视器
2. 记录应用内存占用
3. 进行多轮对话后再次检查

- 空闲时 < 200MB
